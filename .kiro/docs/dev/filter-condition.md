# FilterCondition — AND/OR Filter Logic

`FilterCondition` enables nested AND/OR filter logic for all query endpoints.

**Location:** `libs/shared/shared/data/core/src/filter.rs`

---

## Type Definition

```rust
pub enum FilterCondition {
    And(Vec<FilterCondition>),  // All conditions must match
    Or(Vec<FilterCondition>),   // Any condition can match
    Leaf(FilterEnum),           // Single filter comparison
}
```

---

## API Usage

Clients control filter logic via the `_condition` query parameter:

```
GET /roles?name=eq|admin&status=eq|active                # AND (default)
GET /roles?name=eq|admin&status=eq|active&_condition=or   # OR
```

The `_condition` parameter is auto-generated by the `ParamFilter` macro on every `FilterParams` struct. It defaults to `"and"`.

---

## How It Works

### ParamFilter macro generates:

```rust
#[derive(Deserialize, Debug, Clone)]
pub struct RoleDataFilterParams {
    pub _condition: String,  // "and" or "or", defaults to "and"
    pub name: Option<FilterParam<String>>,
    pub status: Option<FilterParam<String>>,
    // ... other fields
}

impl RoleDataFilterParams {
    pub fn all_filters(&self) -> FilterCondition {
        let mut leaves: Vec<FilterEnum> = vec![];
        // ... collect present filters into leaves
        if self._condition == "or" {
            FilterCondition::Or(leaves.into_iter().map(FilterCondition::Leaf).collect())
        } else {
            FilterCondition::And(leaves.into_iter().map(FilterCondition::Leaf).collect())
        }
    }
}
```

### QueryManager trait accepts FilterCondition:

```rust
fn filter(
    pagination: &Pagination,
    order: &Order,
    filter: &FilterCondition,
) -> impl Future<Output = Result<QueryResult<MD>, DbErr>>;
```

### Query macro recursively builds SeaORM Condition:

```rust
fn build_filter_condition(filter_condition: &FilterCondition) -> Condition {
    match filter_condition {
        FilterCondition::And(conditions) => {
            let mut condition = Condition::all();
            for c in conditions { condition = condition.add(Self::build_filter_condition(c)); }
            condition
        }
        FilterCondition::Or(conditions) => {
            let mut condition = Condition::any();
            for c in conditions { condition = condition.add(Self::build_filter_condition(c)); }
            condition
        }
        FilterCondition::Leaf(filter_enum) => {
            // Column::from_str + filter_condition_column
        }
    }
}
```

## Programmatic Usage

Build complex nested conditions in code:

```rust
use shared_shared_data_core::filter::{FilterCondition, FilterEnum, FilterParam, FilterOperator};

// Simple AND (from Vec)
let condition: FilterCondition = vec![name_filter, status_filter].into();

// Nested AND/OR
let condition = FilterCondition::And(vec![
    FilterCondition::Leaf(name_filter),
    FilterCondition::Or(vec![
        FilterCondition::Leaf(status_active),
        FilterCondition::Leaf(status_pending),
    ]),
]);
// Generates: WHERE name = 'x' AND (status = 'active' OR status = 'pending')

// Push a filter to existing condition
let mut condition = FilterCondition::from(filters);
condition.push_leaf(extra_filter);

// Collect all leaf filters (flattened)
let leaves: Vec<FilterEnum> = condition.collect_leaves();
```

## Conversions

| From | To | Method |
|------|----|--------|
| `Vec<FilterEnum>` | `FilterCondition` | `.into()` or `FilterCondition::from(vec)` |
| `&Vec<FilterEnum>` | `FilterCondition` | `FilterCondition::from(&vec)` |
| `FilterCondition` | `Vec<FilterEnum>` | `.collect_leaves()` (flattens tree) |
